# xiaomiao-paper-digest

微电子 / 集成电路论文晨报静态站点仓库。

这个仓库用于托管每日生成的论文晨报页面，方便通过 GitHub Pages 长期稳定访问，不再依赖容易失效的临时隧道链接。

## 站点地址

- 主入口：<https://xiaomiao-fsd.github.io/xiaomiao-paper-digest/>
- PC 版：<https://xiaomiao-fsd.github.io/xiaomiao-paper-digest/desktop.html>
- 手机版：<https://xiaomiao-fsd.github.io/xiaomiao-paper-digest/mobile.html>

## 仓库用途

这个仓库本身不负责抓取论文，而是承担下面这几个角色：

1. 存放最新一版静态页面
2. 作为 GitHub Pages 的发布源
3. 提供可长期分享的公开链接
4. 让晨报网页和数据结果可以留痕、可回看、可维护

## 当前包含的文件

- `index.html`
  - 自动分流入口页
  - 会根据设备跳转到 PC 或手机版本
- `desktop.html`
  - 面向桌面端阅读
  - 以表格形式展示论文信息
- `mobile.html`
  - 面向手机端阅读
  - 以卡片形式展示论文信息
- `latest.json`
  - 最近一次晨报生成结果的结构化数据
  - 适合程序读取、二次处理或调试
- `README.md`
  - 仓库说明与维护备注

## 页面内容有什么

当前晨报页面会展示：

- 论文标题与原始链接
- 来源期刊 / 平台
- 发布时间
- 作者信息
- 高亮关键词
- English Abstract 原文
- 中文摘要概括
- 基于标题 / 摘要生成的正文阅读导向概括

## 内容来源

论文晨报由本地脚本自动生成，当前会聚合并筛选这些来源：

- arXiv
- Science Advances
- Nature Electronics
- Nature Materials

## 筛选策略

筛选逻辑大致是：

1. 先以**微电子 / 集成电路**作为宽泛召回范围
2. 过滤掉偏**数字电路 / IC 设计**方向的内容
3. 在剩余结果中优先保留：
   - 晶体管 / FET / CMOS / GAA 相关工作
   - 微电子材料相关工作
   - Intel / TSMC 显式提及内容

所以它不是“泛论文订阅页”，而是偏向**微电子材料 + 晶体管器件方向**的定向晨报。

## 更新链路

当前整套链路大致是这样的：

1. 本地定时任务每天早上运行
2. 本地脚本抓取并筛选论文来源
3. 生成：
   - 文本晨报消息
   - HTML 页面
   - JSON 数据
4. HTML / JSON 文件同步到本仓库
5. GitHub Pages 对外发布静态页面
6. 最终得到一个稳定的公开访问链接

## 这个仓库不包含什么

为了让仓库保持干净、轻量和适合公开访问，这里通常**不放**：

- 本地运行状态文件
- 代理配置
- OpenClaw 私有会话信息
- 定时任务内部状态
- 机器环境相关的临时文件

也就是说，这里放的是**发布产物**，不是整套运行环境本身。

## 维护说明

如果以后要继续维护这个仓库，最常见的事情有：

### 1) 页面样式调整

如果想改页面外观，通常改的是本地生成脚本输出的 HTML 模板，而不是直接手改这个仓库里的页面文件。

### 2) 数据来源扩展

如果以后要加入更多期刊 / RSS / API 来源，也应该优先修改本地抓取与筛选脚本，再重新生成并同步到这里。

### 3) 站点访问异常

如果 GitHub Pages 刚开启或刚更新后短时间内出现 404，通常是 GitHub 还在部署，等几分钟再刷新即可。

### 4) 仓库内容和页面不同步

如果仓库内容更新了但页面还没刷新，通常也是 Pages 部署延迟，可以稍等片刻；必要时检查 Pages 构建状态。
